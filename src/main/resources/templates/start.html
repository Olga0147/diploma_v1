Понятия в ООП

Класс

Класс задается:
  • своим типом (именем класса)
  • информацией о суперклассах и реализуемых интерфейсах
  • своими членами (полями, методами и внутренними абстракциями)
Члены класса могут реализовывать свойства объекта (поля и методы экземпляра), а могут реализовывать свойства собственного класса (статические поля и методы).

Для реализации свойства объекта «иметь состояние» в классе существуют атрибуты (поля).В них могут хранится как простые величины (для атрибутов примитивного типа), так и ссылки (указатели) на объекты других классов для ссылочных переменных типов, определяемых пользователем.

Для реализации поведения объекта используются функции-члены класса. Функции-члены экземпляра работают в контексте состояния конкретного объекта и имеют доступ как к членам экземпляра, так и к статическим членам класса. Статические функции-члены реализуют поведение собственно класса и имеют доступ только к статическим членам класса.

Взаимоотношения между классами

Существует три типа отношений между классами:
  • Ассоциация - обозначает наличие логической связи между классами.
  • Агрегация и композиция (на самом деле являются частными случаями ассоциации.)
    • Агрегация — отношение когда один класс является частью другого. Например, Студент входит в Группу любителей физики.
    • Композиция — еще более «жесткое отношение, когда экземпляр класса не только является частью другого экземпляра, но и вообще не может принадлежат еще кому-то.
  • Обобщение/Расширение (наследование)

Объект

Определения по Бучу:
  • объект обладает состоянием, поведением и идентичностью
    • Поведение – это то, как объект действует и реагирует; поведение выражается в терминах состояния объекта и передачи сообщений. Т.е. поведение объекта – это его наблюдаемая и проверяемая извне деятельность.
    • Состояние объекта характеризуется перечнем (обычно статическим) всех свойств данного объекта и текущими (обычно динамическими) значениями каждого из этих свойств.
    • Идентичность – это такое свойство объекта, которое отличает его от всех других объектов.
  • структура и поведение схожих объектов определяет общий для них класс
  • термины «экземпляр класса» и «объект» взаимозаменяемы

Взаимоотношения между объектами

Все отношения между объектами могут быть сведены к двум типам:
  • Ассоциация – отношение, позволяющее реализовать взаимодействие клиент-сервер. При ассоциативном отношении объекты могут существовать независимо друг от друга по времени жизни.
  • Агрегация и композиция – отношение, служащее для определения понятия целое-часть и организации иерархий объектов. При композиционном отношении между объектами существует зависимость по времени жизни: агрегат-часть не может существовать без агрегата-целого.
Другими словами:
  • Ассоциация - Объекты ссылаются друг на друга. При этом они остаются полностью независимыми друг от друга.
  • Агрегация - Объект является частью другого. Агрегат-часть инициализируются вне основного объекта.
  • Композиция - Объект является частью другого. Композит - часть инициализируются внутри основного объекта.
  • Наследования - один класс заимствует, а также расширяет и/или специализирует структуру и функциональный контракт одного или нескольких родительских классов.
  • Реализация - описывает представление абстракции и механизмы достижения желаемого поведения объекта.( интерфейс и реализация - две части любого класса)

Понятия в Системе (Версия 1.1)

Класс

Класс задается:
  • своим типом (идентификатором класса)
  • своими членами (Атрибутами и Шаблонами)
Члены класса могут реализовывать свойства объекта (Атрибуты и Шаблоны экземпляра), а могут реализовывать свойства собственного класса (имя, системное имя, принадлежность к Странице, описание класса).

Для реализации свойства объекта «иметь состояние» в классе существуют Атрибуты (поля).В них могут хранится как простые величины (для атрибутов примитивного типа), так и ссылки на Ресурсы (картинки, видео...).

Для реализации поведения объекта используются Шаблоны Класса. Шаблоны экземпляра работают в контексте состояния конкретного объекта и имеют доступ только к членам экземпляра.

Взаимоотношения между классами

Существует два типа отношений между классами:
  • Ассоциация - обозначает наличие логической связи между классами.
    Например, есть несколько экземпляров Классов:"Информационная доска" и "Контакты". Тогда Шаблон "И.Д" (метод) может включать в себя гиперссылку на получения Шаблона для экземпляра "К." (результа работы метода).
  • Агрегация и композиция
    • Агрегация — отношение когда один класс является частью другого. Доступно только для Страниц.
      Например, есть несколько экземпляров обычных Классов: "Меню", "Приветственная информация" и "Контакты" и экземпляр Класса-Страницы "Главная страница". Тогда Шаблон "Г.С." (метод) может включать в себя вызов Шаблонов (методов) для экземпляров "М.","П.С." и "К."
    • Композиция — еще более «жесткое отношение, когда экземпляр класса является частью другого экземпляра.
      Например, есть несколько экземпляров Классов: "Биография" с Атрибутом типа XMEMO "Описание" и "Города" с Атрибутом типа XMEMO "Текст".Тогда Шаблон "Б." (метод) может включать в себя гиперссылку на получения Шаблона для экземпляра "К." (результа работы метода) с курсором, установленным в определенное место.

Объект

  • объект обладает состоянием, поведением и идентичностью
    • Поведение – это его наблюдаемая и проверяемая извне деятельность, реализованное через Шаблоны.
    • Состояние объекта характеризуется перечнем всех Атрибутов данного объекта и текущими значениями каждого из этих Атрибутов.
    • Идентичность – это такое свойство объекта, которое отличает его от всех других объектов, реализуется идентификатором.
  • структура и поведение схожих Объектов определяет общий для них Класс
  • термины «экземпляр Класса» и «Объект» взаимозаменяемы

Взаимоотношения между объектами

Все отношения между объектами могут быть сведены к двум типам:
  • Экземпляр Ассоциации - Объект имеет ссылку на другой Объект. Экземпляр создается вручную.
    Например, Объект "Меню" при обработке Шаблоном выдаст html-блок с гиперссылками на другие Объекты
  • Экземпляр агрегации и экземпляр композиции
    • Экземпляр агрегации — отношение когда один Объект может включать в себя другие Объекты, но Объекты существуют независимо друг от друга. Доступно только для экземпляров Страниц. Экземпляр создается вручную.
      Используется для контроля вложенности. Доступна степерь вложенности : 2.
    • Экземпляр композиции — еще более «жесткое отношение, когда Объект является частью другого Объекта на уровне данных. Экземпляр создается автоматически, когда Парсер нашел специфическую метку.
      Например, есть Объект "Биография Петра" с Атрибутом типа XMEMO "Описание" и "Города России" с Атрибутом типа XMEMO "Текст".Тогда Атрибут "Описание" содержит фразу "Петр жил в Ростове" со специальной меткой - авторской проводкой на начало фразы "Ростов - это ..." Атрибута "Текст". Таким образом Шаблон "Б." (метод) включит гиперссылку на слово "Ростове" на получения Шаблона для экземпляра "Г." (результа работы метода) с курсором, установленным в слово "Ростов".

Шаблон

Шаблон - это единственный публичный метод. Шаблон из значений полей Объекта и при необходимости, используя информацию из различного вида отношений, формирает некоторый документ (html). Условно можно воспринимать как toString();
Физически Шаблон является некоторым текстом, содержащим разметку html и специальные метки системы,указывающие Парсеру, что делать, например:
  • Вставить данные из конкретного поля Объекта
    @fieldName

  • Вставить результат метода - конкретного Шаблона целевого Объекта (только при зарегистрированной Агрегации и Экземпляре Агрегации, доступно для источника - Объекта Класса-Страницы)
    @aggregationImpl(int aggregationImplId, int templateId,int objectId)
  • Вставить гиперссылку на результат метода - конкретного Шаблона целевого Объекта (только при зарегистрированной Ассоциации и Экземпляре Ассоциации, рекомендуется только для целевых Объектов Классов-Страниц)
    @assosiationImpl(int assosiationImplId, int templateId,int objectId)
  • Вставить гиперссылку с якорем на результат метода - конкретного Шаблона целевого Объекта (только при зарегистрированной Композиции, рекомендуется только для целевых Объектов, которыми пользователь владеет, так как предполагается менять данные Объектов)
    В Атрибуте Объекта с id = 7:
    Он жил в <composition compositionId="5" objectId="10" fragment="nsk">Новосибирске.</composition>
    В Атрибуте Объекта с id = 10:
    <composition compositionId="5" fragment="nsk"/>Новосибирск - это ...

  • Вставить результаты конкретного Шаблона каждого Объекта, имеющего Экземпляр Агрегации конкретной Агрегации.
    @aggregation(int aggregationId, int templateId)
  • Вставить результаты конкретного Шаблона каждого Объекта, имеющего Экземпляр Ассоциации конкретной Ассоциации.
    @assosiation(int assosiationId, int templateId)

Типы Атрибутов

  • Число с плавающей запятой (FLOAT)
  • Число целое (INTEGER)
  • Число целое малое (SMALLINT)
  • Строка длины N (STRING sizeof(N))
  • Текст (TEXT)
  • Текст с специальными вставками (XMEMO)
  • Набор байт (MMEDIA)
  • Время (TIME)
Атрибут типа XMEMO представлен может содержать специальные метки, указывающие на авторские проводки. Только при зарегистрированной Композиции Парсер будет обрабатывать поле особым образом.
Атрибут типа MMEDIA содержит некоторые файлы Объекта, такой как фото, видео или др. Атрибут должен обязательно одержать непосредственный тип контента (jpg, wmv ...).
Любой Атрибут Объекта является приватным полем (принцип инкапсуляции), т.е. нельзя непосредственно получить их значение. Если есть нужда в получении конкретных Атрибутов, то необходимо создать Шаблон, в котором упомянаются только нужные Атрибуты.

Ресурсы

Ресурсы - это статические поля Класса, которые содержат файлы (фото, видео или др.), которые могут быть использованы Объектами Класса - владельца Ресурса.
Например, есть несколько Объектов Класса "Меню", но все они нуждаются в одном и том же картинке - логотипе (Ресурсе Класса "Меню").
