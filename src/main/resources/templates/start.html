Понятия в ООП

Класс

Класс задается:
  • своим типом (именем класса)
  • информацией о суперклассах и реализуемых интерфейсах
  • своими членами (полями, методами и внутренними абстракциями)
Члены класса могут реализовывать свойства объекта (поля и методы экземпляра), а могут реализовывать свойства собственного класса (статические поля и методы).

Для реализации свойства объекта «иметь состояние» в классе существуют атрибуты (поля).В них могут хранится как простые величины (для атрибутов примитивного типа), так и ссылки (указатели) на объекты других классов для ссылочных переменных типов, определяемых пользователем.

Для реализации поведения объекта используются функции-члены класса. Функции-члены экземпляра работают в контексте состояния конкретного объекта и имеют доступ как к членам экземпляра, так и к статическим членам класса. Статические функции-члены реализуют поведение собственно класса и имеют доступ только к статическим членам класса.

Взаимоотношения между классами

Существует три типа отношений между классами:
  • Ассоциация - обозначает наличие логической связи между классами.
  • Агрегация и композиция отношение когда один класс является частью другого. Например, Студент входит в Группу любителей физики. Говорят о композиции, когда есть зависимость по времени жизни
  • Обобщение/Расширение (наследование)

Объект

Определения по Бучу:
  • объект обладает состоянием, поведением и идентичностью
    • Поведение – это то, как объект действует и реагирует; поведение выражается в терминах состояния объекта и передачи сообщений. Т.е. поведение объекта – это его наблюдаемая и проверяемая извне деятельность.
    • Состояние объекта характеризуется перечнем (обычно статическим) всех свойств данного объекта и текущими (обычно динамическими) значениями каждого из этих свойств.
    • Идентичность – это такое свойство объекта, которое отличает его от всех других объектов.
  • структура и поведение схожих объектов определяет общий для них класс
  • термины «экземпляр класса» и «объект» взаимозаменяемы

Взаимоотношения между объектами

Все отношения между объектами могут быть сведены к двум типам:
  • Ассоциация – отношение, позволяющее реализовать взаимодействие клиент-сервер.
  • Агрегация и композиция – отношение, служащее для определения понятия целое-часть и организации иерархий объектов. При композиционном отношении между объектами существует зависимость по времени жизни.

Понятия в Системе (Версия 1.2)

Класс

Класс задается:
  • своим типом (идентификатором класса)
  • своими членами (Атрибутами и Шаблонами)
Члены класса могут реализовывать свойства объекта (Атрибуты и Шаблоны экземпляра), а могут реализовывать свойства собственного класса (имя, системное имя, описание класса).

Для реализации свойства объекта «иметь состояние» в классе существуют Атрибуты (поля).В них могут хранится как простые величины (для атрибутов примитивного типа), так и ссылки на Ресурсы (картинки, видео...).

Для реализации поведения объекта используются Шаблоны Класса. Шаблоны экземпляра работают в контексте состояния конкретного объекта и имеют доступ только к членам экземпляра.

Взаимоотношения между классами

Существует два типа отношений между классами:
  • Ассоциация - обозначает наличие логической связи между классами. Данное отношение создается вручную.
    Например, Класс "Информационная доска" и Класс "Контакты" могут иметь ассоциацию "включает в себя" (Информационная доска включает в себя Контакты).
  • Агрегация — отношение когда один класс является частью другого. Данное отношение создается вручную.
    • Частями Класса "Меню" будут гиперссылки на некоторые Объекты, к которым нужен быстрый доступ.
    • Частями Класса "Страница" будет некоторый Объект Класса "Меню".

Объект

  • объект обладает состоянием, поведением и идентичностью
    • Поведение – это его наблюдаемая и проверяемая извне деятельность, реализованное через Шаблоны.
    • Состояние объекта характеризуется перечнем всех Атрибутов данного объекта и текущими значениями каждого из этих Атрибутов.
    • Идентичность – это такое свойство объекта, которое отличает его от всех других объектов, реализуется идентификатором.
  • структура и поведение схожих Объектов определяет общий для них Класс
  • термины «экземпляр Класса» и «Объект» взаимозаменяемы

Взаимоотношения между объектами

Все отношения между объектами могут быть сведены к двум типам:
  • Экземпляр Ассоциации - Объект имеет ссылку на другой Объект. Данное отношение создается вручную.
    Например, Объект "Меню" при обработке Шаблоном выдаст html с гиперссылками на другие Объекты

  • Экземпляр агрегации и экземпляр композиции
    • Экземпляр агрегации — отношение когда один Объект может включать в себя другие Объекты, но Объекты существуют независимо друг от друга. Данное отношение создается автоматически при проверке поля типа XMEMO, принадлежащего Объекту - источнику.
      Пусть Класс "Меню" содержит поле XMEMO "Адреса". Также у этого Класса есть Объект "Главное Меню". "Главное Меню" в поле "Адреса" содержит XML документ с тегами из предопределенного пространства имен Системы. Эти теги укажут Системе подставить вместо них гиперссылки на некоторые Объекты.

      Пусть Класс "Страница" содержит поле XMEMO "Меню Страницы". Также у этого Класса есть Объект "Новости". "Новости" в поле "Меню" содержит XML документ с тегами из предопределенного пространства имен Системы. Эти теги укажут Системе подставить вместо них Объект "Меню".

Шаблон

Шаблон - это единственный публичный метод. Шаблон из значений полей Объекта и при необходимости, используя информацию из различного вида отношений, формирает некоторый документ (html).
Физически Шаблон является некоторым XML текстом, содержащим разметку html и теги из предопределенного пространства имен Системы,указывающие Системе на необходимость совершить некоторые манипуляции, например:
  • Вставить данные из конкретного поля Объекта
    <field>fieldName</field>

  • Вставить результаты конкретного Шаблона (id = 5) каждого Объекта, имеющего Экземпляр Ассоциации конкретной Ассоциации (id = 2).
    <association associationId="2" templateId="5" type="object"></association>

  • Вставить гиперссылку на конкретный Шаблон (id = 5) каждого Объекта, имеющего Экземпляр Ассоциации конкретной Ассоциации (id = 2).
    <association associationId="2" templateId="5" type="hyperlink" >filed1 filed2 field3</association>

Типы Атрибутов

  • Число с плавающей запятой (FLOAT)
  • Число целое (INTEGER)
  • Число целое малое (SMALLINT)
  • Строка длины N (STRING sizeof(N))
  • Текст (TEXT)
  • Текст с специальными вставками (XMEMO)
  • Набор байт (MMEDIA)
  • Время (TIME)
Любой Атрибут Объекта является приватным полем (принцип инкапсуляции), т.е. нельзя непосредственно получить их значение. Если есть нужда в получении конкретных Атрибутов, то необходимо создать Шаблон, в котором упомянаются только нужные Атрибуты.

Атрибут типа XMEMO

Атрибут типа XMEMO представлен может содержать специальные метки, указывающие на авторские проводки.
  • Вставить гиперссылку (type = "hyperlink") на Объект (id = 7), связанный Агрегацией (id = 2)
    Он жил в <aggregation aggregationId="2" templateId="5" objectId="7" type="hyperlink">Новосибирске.</aggregation>

  • Вставить Объект (type = "objectInTemplate", id = 7) связанный Агрегацией (id = 2)
    <aggregation aggregationId="2" templateId="5" objectId="7" type="object"></aggregation>

Атрибут типа MMEDIA

Атрибут типа MMEDIA содержит некоторый файл Объекта, для него обязательным является указание типа контента:
  • IMAGE - графический файл (jpeg, jpg, gif, bmp...)
  • VIDEO - видеофайл (avi, mpeg...)
  • SOUND - звуковой файл (mp3, wav...)
  • TEXT_FILE - текстовый файл (txt, rtf, doc...)
  • FILE - файл, который не относиться ни к какой из выше перечисленных категорий (zip, exe, xcl...)

Атрибут типа MMEDIA в Шаблон будет вставлен в зависимости от типа контента, а именно:
  • IMAGE - в виде
    <img src="/user-mode/resource/1"></img>
  • VIDEO - в виде
    <video src="/user-mode/resource/1"></video>
  • AUDIO - в виде
    <audio src="/user-mode/resource/1"></audio>
  • TEXT_FILE - в виде
    <a href="/user-mode/resource/1" download=""></a>
  • FILE - в виде
    <a href="/user-mode/resource/1" download=""></a>

Ресурсы

Ресурс - это статическое поле Класса, которое может быть использовано Объектами Класса. Например, есть несколько Объектов Класса "Магазин", и все они нуждаются в одной и той же картинке - логотипе (Ресурсе Класса "Меню").
Ресурс содержит некоторый файл, для которого обязательным является указание типа контента:
  • IMAGE - графический файл (jpeg, jpg, gif, bmp...)
  • VIDEO - видеофайл (avi, mpeg...)
  • SOUND - звуковой файл (mp3, wav...)
  • TEXT_FILE - текстовый файл (txt, rtf, doc...)
  • FILE - файл, который не относиться ни к какой из выше перечисленных категорий (zip, exe, xcl...)
Ресурс в Шаблон будет вставлен в зависимости от типа контента и Системного тэга, а именно:
  • Вставить Ресурс(id = 2) принадлежащий Классу этого Шаблона.
    <resource resourceId="2" type="whole" ></resource>

    • IMAGE - в виде
      <img src="/user-mode/resource/1"></img>
    • VIDEO - в виде
      <video src="/user-mode/resource/1"></video>
    • AUDIO - в виде
      <audio src="/user-mode/resource/1"></audio>
    • TEXT_FILE - в виде
      <a href="/user-mode/resource/1" download=""></a>
    • FILE - в виде
      <a href="/user-mode/resource/1" download=""></a>

  • Вставить Ресурс(id = 2) принадлежащий Классу этого Шаблона для скачивания.
    <resource resourceId="2" type="download" ></resource>

    • Для любого типа - в виде
      <a href="/user-mode/resource/1" download=""></a>

  • Получить ссылку на Ресурс(id = 2) принадлежащий Классу этого Шаблона для самостоятельного конфигурирования.
    <тэг-редактора href="" атрибуты-редактора><resource resourceId="2" type="hyperlink" ></resource></тэг-редактора>
    <тэг-редактора src="" атрибуты-редактора><resource resourceId="2" type="hyperlink" ></resource></тэг-редактора>

    При обработке Шаблона <resource resourceId="2" type="hyperlink" ></resource> будет удалена и в родительском тэге в атрибуте href/src подставиться ссылка на получение файла.
